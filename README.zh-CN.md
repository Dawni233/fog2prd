# Fog2PRD

**从模糊需求到可交付的 PRD — AI 全流程流水线。**

Fog2PRD 是一个 AI Agent，将原始、模糊的产品需求转化为结构化、可直接交付的产出物。它采用**原型优先**工作流：澄清 → 可视化 → 文档 → 交接。

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

---

## 它能做什么

| 阶段 | 做什么 | 产出 |
|------|--------|------|
| **1. 理解需求** | 分析输入，发现歧义点，主动提问澄清 | 需求理解报告 |
| **2. 原型设计** | 生成可交互的 HTML 原型，根据反馈迭代 | 可点击的 HTML 原型 + 业务流程图 |
| **3. 生成文档** | 基于确认后的原型生成正式 PRD | 签字版需求规范文档 |
| **4. 开发交接** | 生成轻量级开发传递单 + 页面线框图 | 开发传递单 + 线框图 |

### 核心设计理念

- **原型优先** — 用户看到界面才能真正确认需求
- **迭代确认** — 原型阶段支持多轮迭代，直到满意
- **双文档输出** — 用户签字版（正式存档）+ 开发传递版（高效交付）
- **5 分钟理解** — 开发版聚焦关键信息

## 工作流程

```
原始需求 → [阶段1: 需求澄清] → [阶段2: 原型与迭代] → [阶段3: 正式PRD] → [阶段4: 开发交接]
```

### 阶段1：需求理解

解析输入，提取角色和功能点，评估复杂度，主动提出澄清问题。

**复杂度评分：**

| 维度 | 简单（1分） | 中等（2-4分） | 复杂（5分） |
|------|------------|--------------|------------|
| 功能点数量 | ≤3 个 | 4-8 个 | >8 个 |
| 用户角色 | 1 个 | 2-3 个 | >3 个 |
| 业务规则 | 简单直接 | 条件分支 | 多层嵌套 |
| 系统集成 | 无 | 单系统 | 多系统 |
| 数据处理 | 基础 CRUD | 计算逻辑 | 复杂算法 |

**评分 → 行动：**
- 1-3 分：快速确认 → 进入原型
- 4-7 分：问答澄清 → 进入原型
- 8-10 分：分步确认 → 进入原型

### 阶段2：原型设计与迭代

生成交互式 HTML 原型，采用现代 UI 模式（卡片、阴影、过渡动画、Lucide 图标）。支持不限次数的迭代直至用户确认。

### 阶段3：生成正式 PRD

将确认后的原型编译为正式 PRD 文档，包含完整功能规格、业务规则、数据规则和审批记录。

### 阶段4：开发交接

生成轻量级开发交接文档，包含：
- 核心需求摘要
- 原型链接
- 业务流程图
- 快速页面线框图（低质量快速模式）

## 使用示例

```
用户: "我需要一个团队任务管理工具..."

Fog2PRD: 📋 先确认几个问题：
         Q1: 团队多少人？
         Q2: 当前工作流最痛苦的地方是什么？
         ...

用户: [回答]

Fog2PRD: 🖥️ 原型已生成，请体验：[原型链接]
         是否符合预期？

用户: "差不多了，侧边栏布局改一下"

Fog2PRD: 🔄 已修改，重新体验：[更新后的链接]

用户: ✅ 确认

Fog2PRD: 📄 PRD 文档已生成：[文档链接]
         📋 开发传递单已生成：[交接文档链接]
```

## 安装

```bash
# 通过 Hermes Agent 安装
hermes install fog2prd

# 或手动克隆
git clone https://github.com/Dawni233/fog2prd.git ~/.hermes/skills/fog2prd
```

安装后，在对话中直接提需求即可：

> *"我们需要一个新的用户引导流程…"*
> *"给仪表盘改版生成 PRD"*
> *"设计移动端 app 的需求"*
