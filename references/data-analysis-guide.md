# 数据需求分析指南

本文档定义了数据字段定义、校验规则和数据字典的编写规范。

---

## 1. 数据分析流程

### 1.1 分析步骤

```
功能需求
    │
    ▼
┌─────────────────┐
│ 1. 识别数据实体  │  从功能中提取核心对象
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. 定义字段属性  │  名称、类型、长度、约束
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 设计校验规则  │  格式、范围、唯一性
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 建立数据关系  │  主外键、关联关系
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. 编写数据字典  │  枚举值、状态码
└─────────────────┘
```

## 2. 数据实体识别

### 2.1 实体识别方法

从功能描述中提取名词，识别核心数据对象：

| 功能描述 | 提取的实体 |
|----------|------------|
| 用户可以创建订单 | 用户、订单 |
| 管理员审批请假申请 | 管理员、请假申请 |
| 系统生成报表 | 报表 |

### 2.2 常见实体类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 主数据实体 | 系统核心业务对象 | 用户、订单、产品 |
| 配置实体 | 系统配置和参数 | 角色、权限、字典 |
| 日志实体 | 操作记录和审计 | 登录日志、操作日志 |
| 关联实体 | 多对多关系的中间表 | 用户角色关联 |

## 3. 字段定义规范

### 3.1 字段属性

| 属性 | 说明 | 示例 |
|------|------|------|
| 字段名 | 中文显示名称 | 用户名 |
| 字段标识 | 数据库字段名 | username |
| 数据类型 | 字段数据类型 | VARCHAR |
| 长度 | 字段长度 | 50 |
| 必填 | 是否允许为空 | 是/否 |
| 默认值 | 默认值 | NULL |
| 说明 | 补充说明 | 用于登录 |

### 3.2 数据类型对照

| 业务类型 | 数据库类型 | 说明 |
|----------|------------|------|
| 短文本 | VARCHAR(50) | 姓名、标题 |
| 长文本 | VARCHAR(500) | 描述、备注 |
| 超长文本 | TEXT | 内容、文章 |
| 整数 | INT | 数量、年龄 |
| 小数 | DECIMAL(10,2) | 金额、比例 |
| 日期 | DATE | 出生日期 |
| 日期时间 | DATETIME | 创建时间 |
| 布尔 | TINYINT(1) | 是否删除 |
| 枚举 | VARCHAR(20) | 状态、类型 |
| JSON | JSON | 扩展数据 |

### 3.3 字段定义模板

```markdown

|--------|------|------|------|------|--------|------|
| 主键ID | id | VARCHAR | 32 | 是 | UUID | 主键 |
| [字段1] | field1 | VARCHAR | 50 | 是 | - | [说明] |

```

## 4. 数据校验规则

### 4.1 校验规则类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 格式校验 | 数据格式验证 | 手机号、邮箱、身份证 |
| 长度校验 | 字符长度限制 | 用户名4-20字符 |
| 范围校验 | 数值范围限制 | 年龄0-150 |
| 唯一性校验 | 数据唯一性 | 用户名不能重复 |
| 存在性校验 | 关联数据存在 | 部门ID必须存在 |
| 条件校验 | 条件性必填 | 选择"其他"时需填写说明 |

### 4.2 校验规则模板

```markdown
### 数据校验规则

| 字段 | 校验规则 | 错误提示 | 校验时机 |
|------|----------|----------|----------|
| [字段] | [规则] | [提示信息] | [时机] |
```

### 4.3 常用校验正则

```markdown
### 常用校验规则

| 校验类型 | 正则表达式 | 说明 |
|----------|------------|------|
| 手机号 | `^1[3-9]\d{9}$` | 11位手机号 |
| 邮箱 | `^[\w-]+(\.[\w-]+)*@[\w-]+(\.[\w-]+)+$` | 标准邮箱格式 |
| 身份证 | `^\d{17}[\dXx]$` | 18位身份证号 |
| 用户名 | `^[a-zA-Z][a-zA-Z0-9_]{3,19}$` | 字母开头，4-20位 |
| 密码 | `^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,20}$` | 8-20位，含字母数字 |
| 邮编 | `^\d{6}$` | 6位数字 |
| IP地址 | `^((25[0-5]|2[0-4]\d|[01]?\d\d?)\.){3}(25[0-5]|2[0-4]\d|[01]?\d\d?)$` | IPv4格式 |
```

### 4.4 校验规则示例

```markdown
### 用户信息校验规则

| 字段 | 校验规则 | 错误提示 | 校验时机 |
|------|----------|----------|----------|
| 用户名 | 长度4-20字符，仅字母数字下划线，字母开头 | 用户名需以字母开头，4-20位字母数字 | 失焦校验 |
| 用户名 | 唯一性校验 | 用户名已存在 | 提交校验 |
| 密码 | 长度8-20字符，必须包含字母和数字 | 密码需8-20位，包含字母和数字 | 失焦校验 |
| 手机号 | 11位手机号格式，唯一性 | 请输入正确的手机号 | 失焦校验 |
| 邮箱 | 邮箱格式（可选） | 请输入正确的邮箱地址 | 失焦校验 |
| 部门ID | 必须存在且未删除 | 请选择有效的部门 | 提交校验 |
```

## 5. 数据关系设计

### 5.1 关系类型

| 关系类型 | 说明 | 示例 |
|----------|------|------|
| 一对一 | 1:1 | 用户-用户详情 |
| 一对多 | 1:N | 部门-用户 |
| 多对多 | M:N | 用户-角色（需要中间表） |

### 5.2 关系定义模板

```markdown
### 数据关系

| 主实体 | 从实体 | 关系类型 | 说明 |
|--------|--------|----------|------|
| [实体A] | [实体B] | 1:N | 一个A对应多个B |
```

### 5.3 关系定义示例

```markdown
### 数据关系

| 主实体 | 从实体 | 关系类型 | 外键 | 说明 |
|--------|--------|----------|------|------|
| 部门 | 用户 | 1:N | dept_id | 一个部门有多个用户 |
| 用户 | 订单 | 1:N | user_id | 一个用户有多个订单 |
| 用户 | 角色 | M:N | - | 通过用户角色关联表实现 |
```
