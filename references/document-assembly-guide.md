# PRD 文档组装指南

本文档定义了将需求分析结果组装为完整 PRD 文档的规则和流程。

---

## 1. 文档组装流程

### 1.1 组装步骤

```
需求理解报告
      │
      ▼
┌─────────────────┐
│ 1. 确定文档结构  │  根据需求复杂度选择章节
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. 填充概述章节  │  背景、目标、范围
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 展开功能需求  │  详细功能描述
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 分析数据需求  │  字段、校验、字典
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. 补充非功能需求│  性能、安全、兼容
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 6. 添加附录      │  术语、参考、变更记录
└────────┬────────┘
         │
         ▼
   PRD 文档输出
```

## 2. 文档结构选择

### 2.1 按复杂度选择结构

#### 简单需求（≤3个功能点）

```markdown
# [项目名称] 需求规格说明书

## 1. 概述
### 1.1 需求背景
### 1.2 业务目标

## 2. 用户角色

## 3. 功能需求
### 3.1 功能清单
### 3.2 功能详细说明

## 4. 原型索引
```

#### 中等需求（4-8个功能点）

```markdown
# [项目名称] 需求规格说明书

## 1. 概述
### 1.1 需求背景
### 1.2 业务目标
### 1.3 适用范围

## 2. 用户角色

## 3. 功能需求
### 3.1 功能清单
### 3.2 功能详细说明

## 4. 数据需求
### 4.1 数据字段定义
### 4.2 数据校验规则

## 5. 非功能需求

## 6. 原型索引

## 附录
```

#### 复杂需求（>8个功能点）

使用完整模板，包含所有章节。

## 3. 各章节填充规则

### 3.1 概述章节

**数据来源**：需求理解报告

| 章节 | 内容来源 | 填充方法 |
|------|----------|----------|
| 需求背景 | 原始需求描述 | 提炼背景和现状问题 |
| 业务目标 | 需求摘要 | 量化目标，设置指标 |
| 适用范围 | 用户角色列表 | 定义系统边界和用户群体 |

**填充示例**：

```markdown
## 1. 概述

### 1.1 需求背景

当前公司员工考勤管理采用纸质登记方式，存在以下问题：
- 数据统计耗时，每月需人工汇总
- 无法实时掌握员工出勤情况
- 容易出现代打卡、漏打卡现象

需要建设电子化考勤系统，提升管理效率。

### 1.2 业务目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 提升统计效率 | 月度考勤统计耗时 | 从2天缩短到2小时 |
| 减少代打卡 | 代打卡投诉率 | 降低90% |
| 实时掌握出勤 | 数据更新延迟 | ≤5分钟 |

### 1.3 适用范围

- 适用对象：公司全体员工（约500人）
- 适用场景：日常上下班考勤、请假审批、加班申请
- 系统边界：与门禁系统对接，不涉及薪资计算
```

### 3.2 用户角色章节

**数据来源**：需求理解报告中的角色列表

**填充规则**：
- 补充角色描述
- 定义权限范围
- 描述典型使用场景

**填充示例**：

```markdown
## 2. 用户角色

### 2.1 角色定义

| 角色 | 描述 | 权限范围 | 使用场景 |
|------|------|----------|----------|
| 普通员工 | 公司在职员工 | 查看个人考勤、提交请假申请 | 日常打卡、请假申请 |
| 部门经理 | 部门负责人 | 查看部门考勤、审批请假 | 审批请假、部门统计 |
| 系统管理员 | HR或行政人员 | 全部权限 | 考勤统计、报表导出、系统配置 |

### 2.2 用户画像

**普通员工**
- 身份特征：公司各岗位员工，技术水平不一
- 核心诉求：快速打卡、便捷请假、查看考勤记录
- 使用频率：每日2次打卡 + 偶尔请假查询
- 技术能力：一般，需要简单易用的界面
```

### 3.3 功能需求章节

**数据来源**：功能点清单 + 功能展开

**填充规则**：
1. 先列出功能清单表格
2. 按功能点逐个展开详细说明
3. 包含：功能描述、前置条件、操作流程、输入输出、业务规则、异常处理、交互说明

**功能排序原则**：
- P0 优先级在前
- 按业务流程顺序排列
- 同模块功能放在一起

### 3.4 数据需求章节

**数据来源**：功能需求中的数据项分析

**填充规则**：
1. 识别所有数据实体
2. 定义字段属性
3. 设计校验规则
4. 建立数据关系
5. 编写数据字典

### 3.5 非功能需求章节

**数据来源**：需求描述 + 默认值

**填充规则**：

```markdown
## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面响应时间 | ≤2秒 | 普通页面加载 |
| 列表查询 | ≤3秒 | 1000条数据以内 |
| 报表生成 | ≤10秒 | 月度统计报表 |
| 并发用户 | ≥100 | 工作日高峰期 |

### 5.2 安全要求

| 要求 | 说明 |
|------|------|
| 身份认证 | 用户名+密码，支持验证码 |
| 密码策略 | 8-20位，含字母数字，定期更换 |
| 权限控制 | 基于角色的访问控制（RBAC） |
| 数据加密 | 密码加密存储，传输使用HTTPS |
| 操作日志 | 记录关键操作，保留6个月 |

### 5.3 兼容性要求

| 类型 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Edge 90+, Firefox 88+ |
| 分辨率 | 1920×1080 及以上 |
| 移动端 | 暂不支持 |
```

### 3.6 原型索引章节

**数据来源**：阶段3输出

**填充规则**：

```markdown
## 7. 原型索引

### 7.1 流程图

- **业务流程图**：[[业务流程图.md]]
- **交互流程图**：[[交互流程图.md]]

### 7.2 页面原型

- **页面线框图**：[[页面线框图.md]]
- **HTML 交互原型**：`prototype/index.html`
```

## 4. 文档格式规范

### 4.1 标题层级

| 层级 | 格式 | 示例 |
|------|------|------|
| 一级 | # 标题 | # 需求规格说明书 |
| 二级 | ## 标题 | ## 功能需求 |
| 三级 | ### 标题 | ### 用户管理 |
| 四级 | #### 标题 | #### 用户添加 |
| 五级 | **粗体** | **前置条件** |

### 4.2 表格格式

- 表头必须有
- 列对齐使用 Markdown 语法
- 复杂表格拆分为多个简单表格

### 4.3 列表格式

- 有序列表用于步骤流程
- 无序列表用于并列项
- 嵌套不超过3层

### 4.4 强调格式

| 格式 | 用途 | 示例 |
|------|------|------|
| **粗体** | 重要内容、字段名 | **用户名** |
| `代码` | 字段标识、编码 | `username` |
| > 引用 | 说明、提示 | > 这是补充说明 |
- [ ] 复选框 | 待确认项 | - [ ] 需确认 |

## 6. 文档命名和存储

### 6.1 文件命名规则

```
[项目名称]-需求规格说明书.md

示例：
- 用户管理系统-需求规格说明书.md
- 考勤系统-需求规格说明书.md
```

### 6.2 存储路径

```
002 领域/产品设计/PRD/[项目名称]-需求规格说明书.md
```

### 6.3 版本管理

| 版本 | 命名规则 | 说明 |
|------|----------|------|
| 初稿 | v1.0 | 首次生成 |
| 修订 | v1.1, v1.2 | 小幅修改 |
| 大改 | v2.0 | 大幅变更 |

文件名可选添加版本号：
```
用户管理系统-需求规格说明书-v1.0.md
```
