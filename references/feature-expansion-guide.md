本文档定义了将功能点展开为详细需求描述的规则和方法。

## 2. 各部分展开方法

### 2.1 功能描述

**格式**：用一句话描述"谁"可以"做什么"，达到"什么目的"

**模板**：
```
[用户角色] 可以通过 [功能名称] [操作行为]，以实现 [业务目标]。
```

**示例**：
```
原始：用户登录
展开：系统用户可以通过用户登录功能输入账号密码验证身份，以获取系统访问权限。

原始：报表导出
展开：系统管理员可以通过报表导出功能将数据统计结果导出为Excel文件，以便离线分析和汇报。
```

### 2.2 前置条件

**识别方法**：
- 访问权限要求
- 数据依赖条件
- 系统状态要求
- 时间/状态限制

**常见前置条件**：

| 类型 | 示例 |
|------|------|
| 权限条件 | 用户已登录、具有管理员权限 |
| 数据条件 | 订单已创建、审批流程已启动 |
| 状态条件 | 任务未完成、账户未冻结 |
| 时间条件 | 在工作时间内、在活动期限内 |

**示例**：
```markdown
**前置条件**：
- 用户已登录系统
- 用户具有"报表管理"权限
- 系统中存在需导出的数据
```

### 2.3 操作流程

**展开方法**：
1. 识别用户入口（菜单、按钮、链接）
2. 描述用户操作动作
3. 说明系统响应
4. 描述结果展示

**流程描述模板**：
```
1. 用户进入 [入口位置]
2. 点击/选择 [操作对象]
3. 系统 [验证/处理逻辑]
4. 展示 [结果/页面]
5. 用户 [后续操作]（可选）
```

**示例**：
```markdown
**操作流程**：
1. 用户进入"用户管理"菜单
2. 点击"添加用户"按钮
3. 系统弹出用户信息填写表单
4. 用户填写表单内容
5. 点击"提交"按钮
6. 系统校验表单数据
7. 校验通过后保存数据
8. 提示"添加成功"
9. 刷新用户列表
```

### 2.4 输入项

**展开维度**：

| 维度 | 说明 | 示例 |
|------|------|------|
| 字段名 | 显示名称 | 用户名 |
| 标识 | 字段ID | username |
| 类型 | 数据类型 | 文本/数字/日期/选择 |
| 必填 | 是否必填 | 是/否 |
| 校验规则 | 验证条件 | 长度、格式、范围 |
| 默认值 | 初始值 | 系统自动生成 |
| 说明 | 补充说明 | 特殊要求 |

**示例**：
```markdown
**输入项**：

| 字段名 | 类型 | 必填 | 校验规则 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| 用户名 | 文本 | 是 | 长度4-20，仅字母数字 | - | 用于登录 |
| 密码 | 密码 | 是 | 长度8-20，含字母数字 | - | 首次登录需修改 |
| 手机号 | 文本 | 是 | 11位手机号格式 | - | 用于找回密码 |
| 邮箱 | 文本 | 否 | 邮箱格式 | - | 接收通知 |
| 角色 | 选择 | 是 | 枚举值 | 普通用户 | 多选 |
| 部门 | 选择 | 是 | 部门树选择 | - | 单选 |
| 状态 | 选择 | 是 | 启用/禁用 | 启用 | - |
```

### 2.5 输出项

**展开方法**：
- 列表展示的字段
- 详情页展示的信息
- 导出文件的内容
- 返回给调用方的数据

**示例**：
```markdown
**输出项**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 用户ID | 文本 | 系统自动生成 |
| 用户名 | 文本 | - |
| 手机号 | 文本 | 脱敏显示：138****1234 |
| 角色 | 文本 | 角色名称列表 |
| 部门 | 文本 | 部门全路径 |
| 状态 | 标签 | 启用/禁用 |
| 创建时间 | 日期 | YYYY-MM-DD HH:mm |
```

### 2.6 业务规则

**规则类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 校验规则 | 数据有效性检查 | 用户名不能重复 |
| 计算规则 | 自动计算逻辑 | 年龄=当前年-出生年 |
| 判断规则 | 条件分支逻辑 | 金额>1000需审批 |
| 关联规则 | 数据依赖关系 | 删除部门前需移除用户 |
| 限制规则 | 数量或范围限制 | 每人每天最多申请3次 |

**规则描述模板**：
```markdown
**业务规则**：

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR001 | 用户名唯一 | 用户名在系统中必须唯一，重复时提示"用户名已存在" |
| BR002 | 密码强度 | 密码必须包含字母和数字，长度8-20位 |
| BR003 | 角色限制 | 至少分配一个角色，最多分配5个角色 |
| BR004 | 部门必填 | 用户必须归属于某个部门 |
| BR005 | 状态联动 | 禁用用户后，该用户立即无法登录系统 |
```

### 2.7 异常处理

**异常类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 输入异常 | 用户输入错误 | 格式错误、必填为空 |
| 业务异常 | 业务规则冲突 | 重复提交、权限不足 |
| 系统异常 | 系统故障 | 网络超时、服务不可用 |
| 数据异常 | 数据问题 | 数据不存在、数据冲突 |

**异常描述模板**：
```markdown
**异常处理**：

| 异常场景 | 错误码 | 提示信息 | 处理方式 |
|----------|--------|----------|----------|
| 用户名已存在 | 1001 | "用户名已存在，请更换" | 清空用户名输入框 |
| 密码强度不足 | 1002 | "密码需包含字母和数字" | 高亮密码输入框 |
| 手机号格式错误 | 1003 | "请输入正确的手机号" | 清空手机号输入框 |
| 无权限操作 | 4003 | "您没有操作权限" | 返回列表页 |
| 系统繁忙 | 5000 | "系统繁忙，请稍后重试" | 显示重试按钮 |
```

### 2.8 交互说明

**描述内容**：
- 页面布局和样式
- 操作反馈和提示
- 动画和过渡效果
- 便捷操作功能

**示例**：
```markdown
**交互说明**：

1. **表单布局**：采用两列布局，左侧基本信息，右侧权限配置
2. **实时校验**：输入框失焦时进行格式校验，错误时显示红色边框和错误提示
3. **密码强度**：密码输入时显示强度指示器（弱/中/强）
4. **角色选择**：使用穿梭框组件，支持搜索和批量选择
5. **部门选择**：使用树形选择器，支持搜索和展开折叠
6. **提交确认**：点击提交前弹出确认对话框，显示填写内容预览
7. **成功反馈**：提交成功后显示成功提示，3秒后自动跳转列表页
```

## 4. 功能点展开示例

### 完整示例：用户添加功能

```markdown
#### F001 用户添加

**功能描述**：系统管理员可以通过用户添加功能创建新用户账号，以赋予新员工系统访问权限。

**前置条件**：
- 管理员已登录系统
- 管理员具有"用户管理"权限
- 系统中已配置部门和角色

**操作流程**：
1. 管理员进入"系统管理 > 用户管理"页面
2. 点击页面上方"添加用户"按钮
3. 系统弹出用户信息填写表单
4. 管理员填写基本信息：用户名、密码、姓名、手机号
5. 选择所属部门（树形选择）
6. 分配用户角色（多选）
7. 设置用户状态（默认启用）
8. 点击"确定"按钮提交
9. 系统校验数据并保存
10. 提示"添加成功"，刷新用户列表

**输入项**：

| 字段名 | 类型 | 必填 | 校验规则 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| 用户名 | 文本 | 是 | 4-20字符，仅字母数字，唯一 | - | 登录账号 |
| 密码 | 密码 | 是 | 8-20字符，含字母数字 | - | 首次登录需修改 |
| 姓名 | 文本 | 是 | 2-20字符，中文或英文 | - | 真实姓名 |
| 手机号 | 文本 | 是 | 11位手机号格式 | - | 找回密码用 |
| 邮箱 | 文本 | 否 | 邮箱格式 | - | 接收通知 |
| 部门 | 选择 | 是 | 必须选择叶子节点 | - | 单选 |
| 角色 | 选择 | 是 | 至少1个，最多5个 | - | 多选 |
| 状态 | 开关 | 是 | 启用/禁用 | 启用 | - |
| 备注 | 文本 | 否 | 最长200字符 | - | - |

**输出项**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 用户ID | 文本 | UUID，系统自动生成 |
| 创建时间 | 日期 | 系统当前时间 |

**业务规则**：

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR001 | 用户名唯一 | 用户名全局唯一，重复时提示"用户名已存在" |
| BR002 | 密码强度 | 必须包含字母和数字，建议包含特殊字符 |
| BR003 | 手机号唯一 | 手机号全局唯一，用于密码找回 |
| BR004 | 角色限制 | 至少分配1个角色，最多5个角色 |
| BR005 | 部门必填 | 用户必须归属于某个部门 |
| BR006 | 默认密码 | 首次登录强制修改密码 |

**异常处理**：

| 异常场景 | 错误码 | 提示信息 | 处理方式 |
|----------|--------|----------|----------|
| 用户名已存在 | 1001 | 用户名已存在，请更换 | 清空用户名，聚焦输入框 |
| 手机号已存在 | 1002 | 该手机号已被使用 | 清空手机号 |
| 密码强度不足 | 1003 | 密码需包含字母和数字 | 高亮密码输入框 |
| 必填项为空 | 1004 | 请填写所有必填项 | 标红未填项 |
| 无权限操作 | 4003 | 您没有操作权限 | 返回列表页 |

**交互说明**：

1. **表单布局**：采用左右两列布局，左列基本信息，右列权限配置
2. **实时校验**：用户名、手机号输入框失焦时进行唯一性校验
3. **密码强度指示**：密码输入时实时显示强度条（弱/中/强）
4. **部门选择器**：支持搜索、展开折叠、单选叶子节点
5. **角色选择器**：使用穿梭框，支持搜索和批量选择
6. **提交确认**：点击确定前弹出二次确认，展示填写内容预览
7. **成功反馈**：添加成功后提示"用户添加成功"，3秒后自动跳转列表页
8. **批量添加**：支持Excel批量导入用户，点击"批量导入"按钮上传文件
```
